# CareerPilot 全职业岗位分析器

本地化运行的全职业岗位分析器，面向数据、产品、运营、研发、咨询、市场、销售、财务、法务、人力、供应链以及可持续发展等多方向求职者。

## 主要功能

- 岗位工作台：合并单条 JD 分析、批量 JD 筛选和行业招聘监测，减少重复入口。
- 投递决策台：单条 JD 分析后优先展示投递建议、关键理由和下一步动作，详细字段折叠查看。
- 评分拆解：按目标偏好、岗位价值、薪资吸引力、风险扣分和简历匹配解释分数来源。
- 投前简历改法：分析目标 JD 后直接提示简历要突出什么、补哪些关键词、少写哪些内容，并给出可改写的摘要和 bullet。
- 低价值风险标签：把低价值岗位拆成职责模糊、纯执行/杂务、销售指标伪装、无转正/培养弱、标题内容不一致等具体风险。
- 行业扩展匹配：目标行业会自动扩展同义词和上位分类，例如人工智能可命中大模型、AIGC、NLP、智能驾驶等表达。
- 城市扩展匹配：支持沪/京/深等城市别名，以及长三角、江浙沪、珠三角、大湾区、京津冀、成渝等城市组。
- JD 导入与分析：支持粘贴文本，以及 PDF / DOCX / TXT / HTML / Excel / 图片上传。
- 批量 JD 分析：支持浏览器插件批量导出、批量粘贴、上传表格/文档和公开链接抓取，逐条输出目标意向匹配度、岗位价值、缺口和投递建议。
- 批量一键视图：可快速切换强投优先、目标城市、目标行业、排除低价值、应届/实习友好和缺口最小队列。
- 今日求职队列：可从单条 JD 或批量结果把岗位加入今日处理清单，在投递管理里跟踪下一步动作和投递状态。
- 省份识别：优先根据城市识别省份，支持省份筛选、远程/多地标注，并用中国省份热力地图展示分布。
- 求职目标卡：集中维护目标说明、求职方向、意向城市、目标行业和岗位关键词；城市支持 285 城市候选和自由输入确认，行业划分对齐招聘网站常见分类，并会参与 JD 匹配加减分。
- 增强匹配算法：装有 rapidfuzz / jieba / scikit-learn 时，会自动启用模糊匹配、中文分词和 TF-IDF 语义相似度。
- 简历工作台：合并 PDF 简历结构化读取、简历匹配、定制简历和不足分析。
- 全局当前简历：左侧边栏可新增、上传 PDF/Word/TXT、选择、编辑、复制、删除简历；内置或默认简历也可改名、修改和删除；上传 PDF 后会自动按教育、经历、项目、技能等板块结构化读取；批量 JD 会默认拿“当前简历”和岗位逐条对比。
- 目标意向：左侧边栏只保留一个目标意向，可编辑名称与内容，也可删除后重新创建。
- 定制简历：根据目标 JD 对母版简历进行摘要、关键词和项目 bullet 改写。
- 行业招聘监测：批量解析招聘文本/链接，标注公司、岗位、薪资、应届生、实习/正式和本次新发现。
- 求职决策：合并 Offer 预测、实习评估和投递管理。
- Offer 预测：结合岗位匹配度、公司层级、学历/英语要求、短板数量和历史投递数据估算简历通过率、面试概率、offer 概率。
- 面经分析：提炼技术问题、行为面问题、英文面试问题和案例题。
- 实习评估：判断实习是否有利于后续全职，输出价值评分、原因、风险点和建议争取的产出。
- 不足与努力方向：按目标岗位类型拆解技能证据、表达协作、资格要求和项目案例，输出更具体的行动项、简历表达和一周计划。
- 投递管理：SQLite 本地数据库记录公司、岗位、薪资、地点、匹配度、投递状态、面试状态和备注。
- 面试与报告：合并面经分析、可视化面板和 Excel/PDF 报告导出。

## 快速启动

第一次使用，双击：

```text
install_and_run.bat
```

以后使用，双击：

```text
start_app.bat
```

如果要让同一 Wi-Fi / 局域网里的其他电脑打开，双击：

```text
start_lan.bat
```

启动窗口会显示类似 `http://192.168.x.x:8503` 的访问地址。其他电脑在浏览器输入这个地址即可打开。请保持启动窗口运行；如果其他电脑打不开，需要在 Windows 防火墙里允许 Python/Streamlit 访问 `8503` 端口。

## 公网部署

如果要让任何电脑、任何网络都能打开，需要部署到公网云平台，而不是只运行本地脚本。推荐把项目上传到 GitHub 后部署到 Streamlit Community Cloud；之后每次把本机修改推送到 GitHub，线上网页会自动更新。

详细步骤见：

```text
DEPLOY.md
```

命令行方式：

```bash
pip install -r requirements.txt
streamlit run app.py
```

一键脚本会创建独立环境 `.venv_cp313`，优先使用 Python 3.13；如果检测到本地环境损坏或 Python 版本过低，会自动进入修复安装流程。

如果项目已经放在 `C:\Code\CareerPilot` 这类英文路径，但 `.venv_cp313` 仍然启动失败，通常是 Python 本体安装在中文用户目录下。建议重新安装 Python 3.13，并把安装目录改成 `C:\Python313`，再回到项目目录双击 `install_and_run.bat`。

## 批量抓取建议

登录态招聘页推荐使用 `browser_extension` 本地浏览器扩展。新版插件支持：

- 单个详情页保存
- 当前列表页批量保存
- 自动滚动后批量保存
- 当前页 + 自动点击下一页批量保存，可设置最多页数和每页滚动次数
- 结构化提取岗位名、公司、薪资、地点、学历和经验

插件导出的文件会按日期进入：

```text
Downloads/CareerPilot_JD/YYYY-MM-DD/

线上部署时，服务端默认会扫描这些目录：

- `项目目录/CareerPilot_JD`
- `项目目录/data/CareerPilot_JD`
- `~/Downloads/CareerPilot_JD`

也可以通过环境变量覆盖：

```text
JD_EXPORT_DIR=/root/Downloads/CareerPilot_JD
```

或配置多个目录：

```text
JD_EXPORT_DIRS=/root/Downloads/CareerPilot_JD;/app/CareerPilot_JD
```
```

Streamlit 的“批量JD分析”里可以按导出日期筛选，避免不同日期的岗位混在一起。

浏览器插件也支持直接上传到云端：

1. 登录 CareerPilot
2. 在侧边栏“浏览器插件云上传”里复制上传地址和上传令牌
3. 打开插件，把两项配置粘进去
4. 之后插件可直接把 JSON 上传到服务器，无需依赖本机 `Downloads`

线上部署建议额外配置：

```text
APP_PUBLIC_URL=http://你的域名或IP:8503
UPLOAD_API_PORT=8765
UPLOAD_API_PUBLIC_URL=http://你的域名或IP:8765/api/plugin-upload
```

## P1 / P2 / P3 含义

- P1：优先投递或必须立即补，和当前目标意向/简历高度相关，最影响投递结果。
- P2：值得投递或短期补，整体方向匹配，但还需要补关键词、项目证据或面试表达。
- P3：可备选或长期提升，重要性低于 P1/P2，适合后续沉淀。

## 说明

- MVP 默认采用本地规则引擎完成关键词提取、岗位分类、匹配分析和缺口分析，不需要服务器或 API key。
- 公开 JD 链接抓取支持多个 URL 并发抓取；可选 Playwright 动态抓取用于公开前端渲染页面。遇到登录、验证码或强风控页面时，优先使用浏览器扩展。
- 图片 OCR 需要本机安装 Tesseract OCR；未安装时可改用文本粘贴。
- 本地投递数据和简历/目标意向数据会自动保存在 `careerpilot.db`。
